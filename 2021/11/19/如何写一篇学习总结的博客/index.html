如何写一篇学习总结的博客

发表于 2021-11-19

写给自己看的注意事项。

写学习型博客的目的是什么：

  1. 记录知识点
  2. 方便以后复习
  3. 展示自己的积累
  4. 让自己真正学懂

以上几个理由都是我写学习类博客的原因，但是最重要的是第四点，要让自己真的弄懂一个知识点，需要输出。

我经常会看一些视频课，专栏或者书籍等，这时候就想要记录点什么，方便复习，也加深理解。这里有一个很容易踩坑的点，这些对于知识点的总结，很容易变成简单的摘抄，”copy-and-paste”一下，拼拼凑凑，一篇文章很容易就出来了，自己的理解却并不多。

不过，既然是学习型的总结，难免会有知识点的记录呀，所以有时摘抄一点也没问题，毕竟也不是写一篇新的论文。

但是一定要注意的是要保证自己理解了！对此，我有几个正在自己身上试验的方法：

  • 写摘要总结，把书读薄
  • 先写标题目录，确保思路清晰
  • 在学习 / 看书 / 读论文的时候，画脑图，记关键词，可以用draw.io或者幕布来做
  • 学习完了之后，提出问题
    • 问题1：这个知识点 / 技术解决了什么问题
    • 问题2：我要如何用（创新）

还有一些注意点：

  • 要明白什么该精读，什么该泛读
  • 读计算机类书籍的话，建议先很快通读一遍
  • 重要的知识点，精读，不求快，可以结合代码去试验
  • 不要“假阅读”，一定要动脑，不要一页停留太久，思维都不知道跑哪儿去了
  • 定下每个任务的预估完成时间，再对比实际完成时间，逐步提高效率

写博客不是为了酷，不是跟着别人写，一定要明白自己想要什么。
