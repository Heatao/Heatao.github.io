rotated sorted array问题

发表于 2019-10-08

记录一下卜老师算法课留的作业

Divide and Conquer

问题描述

​ Suppose an array sorted in ascending order is rotated at some pivot unknown to you beforehand. (i.e., [0, 1, 2, 4, 5, 6, 7] is an ascending array, then it might be rotated and become [4, 5, 6, 7, 0, 1, 2].) How to ﬁnd the minimum of a rotated sorted array?

(Hint: All elements in the array are distinct.)

​ For example, the minimum of the rotated sorted array [4, 5, 6, 7, 0, 1, 2] is 0.

​ Please give an algorithm with O(log n) complexity, prove the correctness and analyze the complexity.

解题思路

描述

​ 注意到一个特点，由于原数组A[n]是递增数组，且元素唯一，则其中任意两个元素a，b，若a<b，则a，b间的元素皆有序。

​ 基于此，采用二分法的思路，将A[n]划分为left，mid，right。

​ 在A[n]中，最小值mini前一位一定大于mini，此时其被划分的两端（无论是l和m还是m和r）一定不会递增。将无序区间再作为子问题迭代下去，直到最简单的情况。

正确性证明

算法基于一点，最小值mini在的划分后的一段数组的两端会无序，

即另一段数组的两端仍然有序，且有序段之间全部有序。

这里给出我自己思考的非严谨的证明过程

分两种情况讨论

mini一开始在数组最左端，数组初始长度为n

  1. mini在有序数组移动后在左半段，此时mini移动距离小于n/2，则原始数组mid位置的移动小于n/2仍然小于n，故划分后新位置上mid~right有序
  2. mini在有序数组移动后在右半段，同理

时间复杂度

将原数组A进行划分，每次以2的倍数进行减少

当n<=3时，T(n) = O(1)

当n>3时，T(n) = T(n/2) + O(n)

二分法的时间复杂度，可求得时间复杂度为O(logn)
